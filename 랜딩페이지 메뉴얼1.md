# 랜딩페이지 시스템 매뉴얼 v1

## 개요

분양 현장별 랜딩페이지를 DB 기반으로 동적 생성하는 시스템.
하나의 템플릿 코드로 여러 현장의 랜딩페이지를 관리.

---

## URL 구조

```
https://onsia.city/{slug}
```

- 예시: `https://onsia.city/cluster-yongin-honorsville`
- 라우트: `src/app/(landing)/[slug]/page.tsx`
- `(landing)`은 라우트 그룹 → URL에 포함 안 됨

---

## 템플릿 #1: 세로 스크롤형 (현재)

분양 랜딩페이지의 가장 일반적인 형태. 이미지를 세로로 쭉 나열하는 방식.

### 페이지 구조 (위→아래 순서)

```
┌─────────────────────────┐
│  1. 히어로 이미지         │  hero_image (풀와이드)
│     + 프로젝트명 오버레이  │  project_name + subtitle
├─────────────────────────┤
│  2. 유튜브 영상           │  youtube_id (선택)
├─────────────────────────┤
│  3. 상단 관심고객 등록 폼  │  개인정보동의 체크박스 포함
├─────────────────────────┤
│  4. 커스텀 섹션           │  sections JSONB (선택)
├─────────────────────────┤
│  5. 이미지 세로 나열       │  gallery[] 전체 풀와이드
│     (20~40장)            │
├─────────────────────────┤
│  6. 하단 관심고객 등록 폼  │  상단과 동일
├─────────────────────────┤
│  7. 푸터                 │  business_info + 면책조항
├─────────────────────────┤
│  8. 하단 고정 바          │  전화 / 카톡 / 방문예약
└─────────────────────────┘
```

|------|------|------|------|

| `developer` | TEXT | N | 시행/시공사 |
| `location` | TEXT | N | 위치 |
| `total_units` | INT | N | 총 세대수 |
| `move_in_date` | TEXT | N | 입주예정일 |
| `phone_number` | TEXT | N | 상담 전화번호 |
| `homepage_url` | TEXT | N | 공식 홈페이지 URL |
| `hero_image` | TEXT | N | 히어로 이미지 URL |
| `logo_image` | TEXT | N | 로고 이미지 URL |
| `gallery` | TEXT[] | N | 갤러리 이미지 URL 배열 (순서대로 렌더링) |
| `sections` | JSONB | N | 커스텀 섹션 배열 (아래 참고) |
| `primary_color` | TEXT | N | 메인 색상 (기본: #1E3A5F) |
| `accent_color` | TEXT | N | 강조 색상 - 버튼 등 (기본: #C9A96E) |
| `youtube_id` | TEXT | N | 유튜브 영상 ID |
| `kakao_chat_url` | TEXT | N | 카카오 오픈채팅 URL |
| `show_bottom_bar` | BOOL | N | 하단 고정 바 표시 여부 (기본: false) |
| `business_info` | JSONB | N | 사업자 정보 (아래 참고) |
| `seo_title` | TEXT | N | SEO 타이틀 |
| `seo_description` | TEXT | N | SEO 설명 |
| `seo_keywords` | TEXT | N | SEO 키워드 (콤마 구분) |
| `og_image` | TEXT | N | OG 이미지 URL |
| `is_published` | BOOL | N | 공개 여부 (기본: false) |
| `view_count` | INT | N | 조회수 |

### sections JSONB 형식

```json
[
  {
    "id": "section-1",
    "type": "custom",
    "title": "프리미엄 입지",
    "content": "<p>HTML 내용</p>",
    "images": ["https://...img1.jpg", "https://...img2.jpg"],
    "order": 1
  }
]
```

- `type`: `hero` | `info` | `gallery` | `location` | `floorplan` | `inquiry` | `custom`
- 현재 템플릿에서는 gallery 위에 렌더링됨

### business_info JSONB 형식

```json
{
  "company_name": "클러스터용인 경남아너스빌 홍보관",
  "registration_number": "000-00-00000",
  "ceo": "홍길동",
  "address": "경기도 용인시 처인구 ...",
  "disclaimer": "본 홍보물은 ... 법적 효력이 없습니다."
}
```

| `page_id` | UUID | 어떤 랜딩페이지에서 왔는지 |

| `utm_campaign` | TEXT | UTM 파라미터 |

## 새 랜딩페이지 추가 방법

### 1단계: 이미지 준비

- 히어로 이미지 1장
- 갤러리 이미지 N장 (세로 나열 순서대로)
- 외부 URL 사용 가능 (Vercel, Supabase Storage, S3 등)
- 이미지 파일명에 한글/공백 있으면 URL 인코딩 필요

### 2단계: DB에 데이터 삽입

**방법 A: 스크립트 사용 (권장)**

`scripts/migration/insert-honorsville-landing.mjs`를 복사해서 수정:

```bash
cp scripts/migration/insert-honorsville-landing.mjs scripts/migration/insert-새현장.mjs
# 파일 내 PAGE_DATA 객체 수정
node scripts/migration/insert-새현장.mjs
```

**방법 B: Supabase 대시보드**

Supabase Dashboard → Table Editor → landing 스키마 → pages 테이블에서 직접 추가

### 3단계: 확인

```
https://onsia.city/{slug}
```

- `is_published: true`여야 접근 가능
- SSG + revalidate 3600초 (1시간 캐시)

## 템플릿 확장 계획

현재는 **세로 스크롤형** 1종만 있음. 향후 추가할 템플릿:

| # | 템플릿명 | 특징 | 적합한 현장 |
|---|---------|------|-----------|
| 1 | **세로 스크롤형** (현재) | 이미지 풀와이드 나열 | 일반 분양 현장 |
| 2 | 섹션 카드형 | 카드 레이아웃 + 애니메이션 | 프리미엄 브랜드 |
| 3 | 원페이지 인터랙티브 | 스크롤 애니메이션 + 패럴랙스 | 대형 랜드마크 |
| 4 | 미니멀 정보형 | 텍스트 중심 + 핵심 정보 | 오피스텔/상가 |

### 템플릿 전환 방법 (향후)

`pages` 테이블에 `template_type` 컬럼 추가 예정:
```sql
ALTER TABLE landing.pages ADD COLUMN template_type TEXT DEFAULT 'scroll';
```

`page.tsx`에서 template_type에 따라 다른 컴포넌트 렌더링:
```typescript
if (page.template_type === 'card') return <CardTemplate page={page} />
if (page.template_type === 'interactive') return <InteractiveTemplate page={page} />
return <ScrollTemplate page={page} />  // 기본값
```

## 주의사항

1. **이미지 URL**: 외부 이미지 사용 시 CORS 및 접근성 확인 필요
2. **slug 규칙**: 영문 소문자 + 숫자 + 하이픈만 사용 (한글 X)
3. **기존 라우트 충돌**: `/about`, `/admin`, `/properties` 등 기존 라우트와 같은 slug 사용 금지
4. **캐싱**: `revalidate = 3600` (1시간). 즉시 반영 필요 시 Vercel에서 수동 revalidate
5. **환경변수**: `NEXT_PUBLIC_SUPABASE_CRM_URL`, `NEXT_PUBLIC_SUPABASE_CRM_ANON_KEY` 필요
6. **개인정보**: 문의 폼에 개인정보 수집동의 체크박스 필수 포함
